## 今日学习内容

### 1. Linux 用户

Linux 系统主要分为三类用户：

| 用户类型 | 说明 |
|----------|------|
| root | 超级管理员，拥有最高权限 |
| 普通用户 | 日常开发使用的用户 |
| 系统用户 | 软件运行时创建的用户，如 mysql、redis 等 |

---

### 2. 查看文件权限

查看文件权限：

```bash
ls -l
```

示例：

```text
-rwxr-xr-- 1 liuyu users 256 Jul 2 test.txt
```

权限信息主要包含：

- 文件类型
- 文件权限
- 文件拥有者
- 所属用户组
- 文件大小
- 修改时间
- 文件名称

---

### 3. Linux 权限

Linux 权限由三个字符组成：

| 权限 | 含义 |
|------|------|
| r | Read（读取） |
| w | Write（写入） |
| x | Execute（执行） |

例如：

```text
rw-
```

表示：

- 可读
- 可写
- 不可执行

例如：

```text
r-x
```

表示：

- 可读
- 不可写
- 可执行

---

### 4. 三组权限

Linux 权限共分为三组：

```text
-rwxr-xr--
```

可拆分为：

```text
rwx   r-x   r--
```

分别表示：

| 权限组 | 说明 |
|--------|------|
| Owner | 文件拥有者 |
| Group | 所属用户组 |
| Others | 其他用户 |

---

### 5. chmod —— 修改权限

增加执行权限：

```bash
chmod +x test.txt
```

移除执行权限：

```bash
chmod -x test.txt
```

增加写权限：

```bash
chmod +w test.txt
```

---

### 6. 数字权限

Linux 中常用数字表示权限：

| 权限 | 数值 |
|------|------|
| r | 4 |
| w | 2 |
| x | 1 |

计算规则：

| 权限 | 数值 |
|------|------|
| rwx | 7（4+2+1） |
| rw- | 6（4+2） |
| r-x | 5（4+1） |
| r-- | 4 |
| --- | 0 |

例如：

```bash
chmod 755 test.txt
```

表示：

| 用户 | 权限 |
|------|------|
| Owner | rwx |
| Group | r-x |
| Others | r-x |

---

### 7. 常见权限

| 权限 | 说明 |
|------|------|
| 777 | 所有人拥有全部权限（开发环境可见，生产环境慎用） |
| 755 | 拥有者全部权限，其余用户可读、可执行（目录和脚本最常见） |
| 644 | 拥有者可读可写，其余用户只读（普通文件最常见） |
| 600 | 仅拥有者可读可写（私密文件） |

---

### 8. chown —— 修改拥有者

修改文件拥有者：

```bash
sudo chown root test.txt
```

修改拥有者和所属用户组：

```bash
sudo chown liuyu:developers test.txt
```

---

### 9. sudo

普通用户执行管理员命令时，可以使用：

```bash
sudo 命令
```

例如：

```bash
sudo apt update
```

sudo 表示临时获得管理员权限。

---

### 10. 企业实际应用

当执行脚本时：

```bash
./spark-start.sh
```

如果提示：

```text
Permission denied
```

通常是因为脚本没有执行权限。

解决方法：

```bash
chmod +x spark-start.sh
```

再次执行即可。

这是企业开发中最常见的问题之一。
